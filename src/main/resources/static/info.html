About This Billing Dashboard

Overview

The Billing Dashboard is a focused, production-oriented interface for exploring large telecom billing datasets. It provides fast, paginated access to records, real-time filtering and ranking, and pre-built analytics that surface charge concentrations, outliers, and trends across accounts and departments. Built with scalability in mind, the dashboard demonstrates how ETL-driven data can be transformed into actionable operational insights for finance and operations teams.

What You Can Do Here
  • Load a paginated summary of billing records with on-demand filters
  • Browse individual telecom records in a sortable, searchable table
  • Filter and segment records by department, state, or billing period
  • Identify top-charging accounts with configurable Top‑N queries
  • Explore charge patterns and trends across accounts and departments
Technologies Used
  • Java 21 – Core language used to implement the ETL pipeline, service layer, model layer, and backend logic.
  • Spring Boot (Web) – Framework for building the REST API, handling dependency injection, and structuring the application.
  • AWS – Infrastructure and services used in production: ECS, RDS, CloudWatch, SQS, SNS, S3.
  • Docker – Containerization for consistent deployment environments.
  • PostgreSQL – Production relational datastore used for billing records.
  • JUnit 5 & Mockito – Testing frameworks for unit and integration tests.
  • Maven – Build and dependency management tool used to manage libraries, run tests, and package the application.
  • HTML5 + Bootstrap – Frontend UI for viewing analytics, summaries, and interactive filtering of billing records.
